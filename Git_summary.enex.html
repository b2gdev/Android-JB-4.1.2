git config --global user.name "Example Surname"

git config --global color.branch auto

git config --global core.editor gedit


To view the configs

TCBIN Work flow

1. Clone repo
# git clone [source] [destination]
git clone username@GIT-SERVER.Zone24x7.lk:/repos/jb/tcbin_jb.git .

ICS source: GIT-SERVER.Zone24x7.lk:/repos/ics/tcbin_revb_ics.git
JB source: GIT-SERVER.Zone24x7.lk:/repos/jb/tcbin_jb.git

To view the commit messages use, git log.

2. Checkout and track the development branch
     
Note: All dev work will be done on the "development" branch, while the releases will be in the "master" branch
git checkout --track origin/development


3. Create and checkout a new branch for local dev work
git checkout -b ruvindad-wifi     
#To check the currently active branch
git branch


4. Do changers

     Note: Branching and switching between branches is very cheap.
     Note: Use can use nested branches to organize and manage your changers. 
     

5. check changers
# to check changers
git status


6. Add to staging area (take snapshot of changers)
git add [file name]

# Add all (files and directories) to the Git repository
git add .

# register all file/folder deletion
git add -A .

Note: Recompiled binaries will be detected as a change even though its source was not changed. If you didn't edit the source, then don't commit its recompiled binary

To permanently ignore changers to particular files/folders
add rule to .gitignore file in source root



7. Commit locally (commit snapshot)
git commit -m "msg"


8. Repeat from (4) - (7)


9. Backup local feature dev branch to git server
git push origin ruvindad-wifi


10 . Repeat from (4) - (9)


11.  Committing feature to main development branch
git checkout development

git pull origin development

git checkout ruvindad-wifi 

git rebase development 
# If any conflicts are found
     # Open the conflicted source files and edit the conflict lines
     # "git add" the binaries
     # "git add [conflicted files]" and then "git rebase --continue"     
     # Recompile and add the merged binaries as a separate commit after the rebase
              
# if you want to merge/edit all your commit messages 
git rebase -i development or git rebase -i HEAD~n       [where n is the number of commits to merge]
     # follow the instructions that git provides
     # if -i fails, use "git rebase --abort" to cancel all comment editing

git checkout development
git merge ruvindad-wifi 
git push origin development

# delete feature branch/s if needed
git branch -d ruvindad-wifi
git push origin :ruvindad-wifi


12. Repeat from (3) - (11)


13. Release
# Merge development branch into master
git checkout master
git pull origin master
git merge development 
git push


# Tag the release
git tag 1.0.0 

#Push up your changes
git push --tags

Note: if a bug is found in the release, make the appropriate changers to directly to the master branch. Then commit and tag it. Finally merger the changers to the development branch.

Note: can push with --force flag; but use it with caution.

Note: you can use tags for other branches as well.

Other useful undoing commands

If you create files in your working copy which you do not want to commit, you can discard them.

# Make a dry-run to see what would happen # -n is the same as --dry-run 
git clean -n 

# Now delete 
git clean -f

You can check out older revisions of your source code via the commit ID. The commit ID is shown if you enter the git log command. It is displayed behind the commit word.

# Get the log
git log

# Copy one of the older commits and checkout the older revision via 
git checkout commit_name

If you deleted a file but you have not yet added it to the index or committed the change, you can check out the file again.

# Delete a file test01 
git checkout test01

If you added a file to the index(staged it) but do not want to commit the file, you can remove it from the index via the git reset file command.

git reset incorrect.txt

If you deleted a directory and you have not yet committed the changes, you can restore the directory via the following command:

git checkout HEAD -- your_dir_to_restore

to merge 'n' number of commits on the current branch
git rebase -i HEAD~n

Undo changers made since last commit to a particular file

git checkout -- file.fl


Undo changers made since last commit to all files in a particular folder

git checkout -- file/path/


To change commit message

git commit -m "worng msg"
git commit --amend -m "correct msg"


Add files to last commit

git add forgotten_file/s
git commit --amend

To remove files that were wrongly staged
git rm --cache [path/filename] : To remove from cache
git rm -f [path/filename]          : To remove from cache and from disk
